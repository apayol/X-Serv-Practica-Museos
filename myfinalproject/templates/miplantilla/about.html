Información sobre "Museos de Madrid"
La aplicación web descarga información sobre los museos de la ciudad, disponibles en el portal de datos abiertos de Madrid y los ofrece a los usuarios para que puedan seleccionar sus favoritos y comentar sobre aquellos que elijan.


Arquitectura general
Se ha construido como un proyecto Django/Python3, con aplicaciones Django, usando SQLite3 con tablas definidas en modelos de Django.

Se usa Django "Admin Site" para la creación de cuentas a los usuarios en el sistema, y para la gestión general de las bases de datos necesarias.

Se ha utilizado una plantilla Django para definir las páginas que se sirven a los navegadores de los usuarios. Estas plantillas incluyen: un banner, una caja de login o logout, una barra de menú de opciones y un pie de página.

Se ha usado una hoja de estilo CSS para determinar la apariencia, en la cual se especifican: colores y tamaños de letra y el color de fondo (entre otros) de los elementos que se definieron en los HTML de plantilla.

Se descargan los datos de los museos in situ del portal de datos abiertos de Madrid (enlace en el pie de página).

Funcionamiento general
Los usuarios son dados de alta desde el "Admin Site" de Django. Una vez dados de alta, son considerados "usuarios registrados".

El listado de museos se cargará a partir del fichero XML cuando un usuario indique que se carguen. Hasta que un usuario no indique que se carguen por primera vez, no habrá listado de museos en la base de datos de la aplicación.

Los usuarios registrados pueden crear su selección de museos favoritos. Por ello, disponen de una página personal, "página del usuario".

Cualquier navegador puede acceder a la interfaz pública del sitio, que ofrece la página personal de cada usuario, para cualquier usuario del sitio.

Cualquier usuario podrá indicar que quiere una vista del sitio que solo incluya los Museos (accesibilidad = "1").

Funcionalidad
Interfaz pública: La página principal (/) tiene un listado de museos y otro con enlaces a páginas personales.

Muestra un listado con los 5 museos con más comentarios (si no hubiera 5 se muestran los que sí tengan comentarios) y aparece el nombre del museo (hipervínculo que enlaza a la URL del museo en el portal esmadrid), su dirección y un enlace, "Más información, que apunta a la página del museo en la aplicación.

Muestra un listado (columna lateral) con enlaces a las "páginas de usuarios" disponibles. Para cada página personal se muestra el título que le ha dado cada usuario (a modo de enlace a su página personal) y el nombre de usuario. Si aún no tiene un título, se le asigna provisionalmente: "Página de id_usuario".

También se muestra un botón que al pulsarlo se verán todos los museos accesibles y solo esos (volviendo a ver todos los museos al pulsar de nuevo).

La página de usuario (/usuario) muestra los museos seleccionados por ese usuario de 5 en 5, separándolos en páginas si es necesario, y mostrando la fecha en que fueron marcados por el usuario.

La página todos (/museos) con todos los museos, apareciendo únicamente el nombre, y un enlace a su página. En la parte superior de la página se pueden filtrar (mediante un formulario desplegable) los museos según el distrito en el que se encuentran.

La página de un museo (/museos/id) muestra toda la información posible del XML del portal del Ayuntamiento además de todos los comentarios que tenga este museo.

La página de usuario xml (/usuario/xml) muestra un documento xml con una entrada para cada museo seleccionado por ese usuario.

La página de información (/about) en la que se encuentra ahora, con la descripción del funcionamiento y autoría.

Interfaz privada: solo para los usuarios registrados, incluye cualquier funcionalidad de la interfaz pública además de, en lapágina de un museo (/museos/id) un formulario para poner comentarios sobre ese museo (comentarios anónimos aunque solo de usuarios registrados).

En lapágina de usuario (/usuario) un formulario para cambiar el estilo CSS de todo el sitio para ese usuario (tamaño de letra y color de fondo). Si se cambian estos valores, quedará cambiado para todas las páginas del sitio para ese usuario. Y un formulario para elegir el título de su página personal.

Otros añadidos: Favicon del sitio en todas las páginas, slider de imágenes y citas ilustres, diseño del banner, XML de página principal, JSON de página principal, JSON de página de usuario, RSS de comentarios (enlace desde inicio), funcionalidad de registro de nuevos usuarios, video insertado en HTML5 en el about con la explicación de funcionalidad.
